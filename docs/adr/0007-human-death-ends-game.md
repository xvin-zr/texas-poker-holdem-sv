# 人类死亡提前结束整局

原胜利判定铁律是「整局胜利仅由存活人数判断」：开枪后若只剩 1 名存活玩家则进 `won`，否则继续下一手。但人类玩家死亡后继续让 AI 互打没有交互价值，且会让玩家困惑「我已经死了为什么还在推进牌局」。

决定：新增 `human-dead` 状态作为受控例外。`won` 仍只表示「唯一存活者诞生」；`human-dead` 表示「人类死亡，整局提前结束」。首个落地点是非 All-in 弃牌开枪：若死亡者是人类，先于 `winIfOnlyOneAlive` 短路到 `human-dead`，即使仍有 ≥2 名 AI 存活。AI 死亡路径不变：≥2 存活进 `hand-resolved`，≤1 存活进 `won`。

`human-dead` 保留最后一手的 `showdown` / `allInSettlement` / `handResolution`，供 UI 继续展示死因；不响应 `next-hand`，只能通过 `start-game` 进入下一局。UI 用阻塞死亡弹窗展示「你已死亡 / 本局已结束」和最后一次开枪结果，按钮「下一局」复用既有重置逻辑。

这是对胜利铁律的明确例外：难以逆转（状态机与 UI 都要区分终局原因）、无上下文会困惑（`won` 不再能覆盖所有终局）、但权衡真实（玩家死亡后继续模拟 AI 局没有价值）。
